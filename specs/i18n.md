# Internationalization

Supported locales are exactly `fi` and `en`.

## Routing

- Finnish is the default locale.
- Routes are prefixed with `/fi` or `/en`.
- Missing or unsupported locales redirect to `/fi`.
- The language chooser preserves the equivalent route when possible.

## Content

Unit and resource names require Finnish and English. Descriptions, addresses,
and reservation instructions support both languages and may fall back to Finnish
when a translation is missing.

## UI Messages

All interface text lives in:

- `apps/web/messages/fi.json`
- `apps/web/messages/en.json`

Tests must ensure both files contain the same message keys.
